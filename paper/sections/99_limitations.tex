\begin{itemize}

\item \textbf{Two agent surfaces.} The crossed
$3\,{\times}\,2\,{\times}\,2$ design exercises Claude Code
(\texttt{claude-sonnet-4-6}) and Codex CLI (\texttt{gpt-5.5}); GPT-5,
Gemini~2.5, and open-weight scaffolds are out of scope. The
three-mechanism account (\S\ref{sec:discussion}) predicts where each
component should generalise, but the joint four-cell footprint is
verified only on these two agents.

\item \textbf{Sample size.} Per-cell $n$ is $93$ for Artifact and $100$
for SWE-bench Mini, three seeds each. The SWE-bench/Claude cost gap of
$\respct{paired_contrasts}{swebench:claude:onlycode-vs-baseline:cost_adj:mean_delta}{swebench:claude:onlycode-vs-baseline:cost_adj:mean_b}$
($p = \resp{paired_contrasts}{swebench:claude:onlycode-vs-baseline:cost_adj:wilcoxon_p}$)
is directional rather than significant; SWE-bench-Verified-scale
replication is reserved for follow-up work.

\item \textbf{Artifact-suite scope.} Tasks are single-language (Python)
on Linux, skewed toward analytical and ML-flavoured computation; the
grader contract in \S\ref{sec:method-artifact} has not been exercised
on Rust, systems-engineering tasks, or long-horizon scenarios.

\item \textbf{Cost is cache-adjusted, not cache-isolated.} Neither the
Anthropic Messages API nor the OpenAI Responses API exposes a
documented knob to disable or partition the server-side prompt cache
(\S\ref{sec:method-cost}). We charge every token at the non-cached rate
and floor each instance's first-turn \texttt{cache\_read} at the per-arm
median; within-task multi-turn cache reads and deeper server-side state
are not adjusted.

\item \textbf{Single-shot, internal-surface scope.} Each
$(\text{instance}, \text{arm}, \text{seed})$ triple is a single agent
invocation; multi-shot prompting, self-consistency, scaffold evolution,
and external MCP tool surfaces (Drive, Slack, vendor APIs;
\S\ref{sec:related}) are out of scope.

\end{itemize}
